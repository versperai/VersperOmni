# 强化学习 — DPO 数据与训练

> 对应脚本：`src/versper/trainer/pretrain.py`（基于 PretrainDataset/SFTDataset 的数据加载）
> 完整的 DPO 偏好训练仍需自定义 loss 实现

## 概述

直接偏好优化（Direct Preference Optimization, DPO）是一种无需显式奖励模型的强化学习方法，通过在偏好数据上直接优化策略来对齐人类偏好。相比 PPO，DPO 更简洁、训练更稳定，是目前对齐微调的主流方法。

DPO 的核心思想：给定 `chosen`（偏好响应）和 `rejected`（非偏好响应）对，模型学习最大化两者在策略下的对数概率差距，从而使模型倾向于生成人类偏好的输出。

## 数据格式

DPO 数据集为 `dpo.jsonl`，采样自 [DPO-En-Zh-20k](https://huggingface.co/datasets/llamafactory/DPO-En-Zh-20k) 并按本项目格式重组。

```json
{
  "chosen": [
    {"content": "什么是机器学习？", "role": "user"},
    {"content": "机器学习是人工智能的一个分支，它使计算机能够从数据中学习和改进...", "role": "assistant"}
  ],
  "rejected": [
    {"content": "什么是机器学习？", "role": "user"},
    {"content": "我不知道。", "role": "assistant"}
  ]
}
```

- `chosen`：高质量、符合人类偏好的完整对话
- `rejected`：低质量或不符合偏好的对应对话
- 每条数据包含两个完整的对话数组，其 user 输入部分相同，assistant 回答质量不同

其他 RL 数据通常保持与 SFT 数据相同的格式，按总长度和对话轮次从 SFT 数据中筛选得到，最后一个 assistant 位置留空用于 rollout 阶段的续写。

## DPO 损失函数简述

DPO 优化的目标函数为：

```
L_DPO(π_θ) = -E[ log σ(β * (log π_θ(chosen) / π_ref(chosen) - log π_θ(rejected) / π_ref(rejected))) ]
```

其中 `β` 控制偏好约束强度，`π_ref` 为参考策略（通常是 SFT 阶段的模型）。该损失函数隐式地增大 chosen 响应的概率同时减小 rejected 响应的概率。

## 训练脚本

### 基于 PretrainDataset 的 DPO 数据加载（当前实现）

当前代码库使用 `PretrainDataset` / `SFTDataset` 加载 DPO 数据的 chosen 和 rejected 部分，分别计算语言建模损失。完整的 DPO loss（偏好损失）需要自定义 loss 实现。

```bash
python -m versper.trainer.pretrain \
    --data_path ../dataset/dpo.jsonl \
    --from_weight sft \
    --learning_rate 1e-6 \
    --epochs 1
```

## Omni 场景下的 DPO 挑战

在多模态 Omni 模型中应用 DPO 比纯文本场景更为复杂：

| 挑战 | 说明 |
|------|------|
| **语音输出评估** | 需要 ASR 系统将生成语音转录为文本，再评估回答质量 |
| **说话人相似度** | 语音输出需保持说话人特征一致性，缺乏客观度量指标 |
| **多模态偏好标注** | 图文+语音的偏好对人工标注成本远高于纯文本 |
| **奖励欺骗（Reward Hacking）** | 模型可能学会利用评估指标的漏洞而非真正提升质量 |

## 最佳实践

- **从干净的 SFT 模型开始**：DPO 训练需要 SFT 模型已经具备基本的对话能力，不要在未微调的预训练模型上直接运行 DPO
- **使用较小的学习率**：`1e-6` 或更低，过大的学习率会导致奖励欺骗或模型崩溃
- **监控奖励差异**：跟踪 chosen 和 rejected 之间的对数概率差异，确保模型在学习而不是记忆
- **限制训练轮数**：1~2 轮足够，过多轮次可能导致过拟合到偏好数据
- **定期回滚**：保存每个 epoch 的 checkpoint，如果发现生成质量下降可回滚到上一版本

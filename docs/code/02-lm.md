# 核心语言模型 -- MiniMindModel & MiniMindForCausalLM

## 架构总览

VersperOmni 的语言模型分为两层：

1. **MiniMindModel** (`model.py:17`) -- 纯 Transformer 骨干网络，不含 LM 头
2. **MiniMindForCausalLM** (`model.py:91`) -- 封装 MiniMindModel，添加 LM 头和生成能力

```
MiniMindForCausalLM (PreTrainedModel + GenerationMixin)
  └── MiniMindModel (nn.Module)
        ├── embed_tokens (nn.Embedding, vocab_size→hidden_size)
        ├── dropout
        ├── layers (MiniMindBlock × 8)
        │     ├── Block 0: Attn → FFN
        │     ├── Block 1: Attn → FFN
        │     ├── ...
        │     └── Block 7: Attn → FFN
        └── norm (RMSNorm)
  └── lm_head (nn.Linear, hidden_size→vocab_size, tied)
```

## MiniMindModel -- Transformer 骨干

### 初始化

```python
class MiniMindModel(nn.Module):
    def __init__(self, config: MiniMindConfig):
        self.embed_tokens = nn.Embedding(config.vocab_size, config.hidden_size)
        self.dropout = nn.Dropout(config.dropout)
        self.layers = nn.ModuleList([
            MiniMindBlock(l, config) for l in range(num_hidden_layers)
        ])
        self.norm = RMSNorm(config.hidden_size, eps=config.rms_norm_eps)
        # 预计算 RoPE 频率
        freqs_cos, freqs_sin = precompute_freqs_cis(...)
        self.register_buffer("freqs_cos", freqs_cos, persistent=False)
        self.register_buffer("freqs_sin", freqs_sin, persistent=False)
```

### 前向传播

```python
def forward(self, input_ids, attention_mask, past_key_values, use_cache):
    # 1. Token Embedding
    hidden_states = self.dropout(self.embed_tokens(input_ids))
    
    # 2. 位置编码 (从 past_key_values 计算 start_pos)
    position_embeddings = (freqs_cos[start_pos:start_pos+seq_len],
                           freqs_sin[start_pos:start_pos+seq_len])
    
    # 3. 逐层计算
    for layer, past_kv in zip(self.layers, past_key_values):
        hidden_states, present = layer(hidden_states, position_embeddings,
                                       past_key_value=past_kv, use_cache=use_cache)
    
    # 4. 最终归一化
    hidden_states = self.norm(hidden_states)
    
    # 5. 汇总 MoE 辅助损失
    aux_loss = sum(l.mlp.aux_loss for l in self.layers if MOE)
    
    return hidden_states, presents, aux_loss
```

### RoPE 恢复机制

当模型在 meta device 上初始化后加载权重时，预计算的 RoPE buffer 会丢失。`forward` 中检测 `freqs_cos[0, 0] == 0` 并自动重新计算。

### 前向传播

```python
def forward(self, input_ids, attention_mask, past_key_values, use_cache,
            logits_to_keep=0, labels=None):
    # 1. 骨干网络前向
    hidden_states, past_key_values, aux_loss = self.model(...)
    
    # 2. LM Head (支持 logits_to_keep 优化)
    slice_indices = slice(-logits_to_keep, None)  # 只计算最后 N 个 token
    logits = self.lm_head(hidden_states[:, slice_indices, :])
    
    # 3. 损失计算 (仅对 labels != -100 的位置)
    if labels is not None:
        shift_logits = logits[..., :-1, :]
        shift_labels = labels[..., 1:]
        loss = F.cross_entropy(shift_logits, shift_labels, ignore_index=-100)
    
    return MoeCausalLMOutputWithPast(
        loss=loss, aux_loss=aux_loss,
        logits=logits, past_key_values=past_key_values,
        hidden_states=hidden_states,
    )
```

### 自定义 Generate 方法

`model.py:144` 实现了一个完整的自回归生成方法，支持以下特性：

| 特性 | 实现 |
|------|------|
| Top-K 采样 | `torch.topk(logits, top_k)` 过滤 |
| Top-P (Nucleus) 采样 | 累积概率超过 top_p 的 token 被过滤 |
| Temperature 缩放 | `logits / temperature` |
| Repetition Penalty | 对已出现 token 的 logits 进行惩罚 |
| KV Cache | 通过 `past_key_values` 增量解码 |
| Streaming | 通过 `streamer.put()` / `streamer.end()` 接口 |
| 多序列生成 | `num_return_sequences` 重复输入 |
| KV Cache 导出 | `return_kv=True` 在 kwargs 中返回 `{"generated_ids": ..., "past_kv": ...}` |

核心采样循环：

```python
for _ in range(max_new_tokens):
    outputs = self.forward(input_ids[:, past_len:], ...)
    logits = outputs.logits[:, -1, :] / temperature
    
    # Repetition penalty
    # Top-K filtering
    # Top-P filtering
    
    next_token = torch.multinomial(softmax(logits), 1)  # 采样
    input_ids = torch.cat([input_ids, next_token], dim=-1)
```

## MiniMindBlock -- 单层结构

定义在 `modules/block.py:8`，采用 Pre-Norm + Residual 结构：

```python
class MiniMindBlock(nn.Module):
    def forward(self, hidden_states, position_embeddings, ...):
        # Pre-Norm Attention + Residual
        residual = hidden_states
        hidden_states = self.input_layernorm(hidden_states)
        hidden_states, present = self.self_attn(hidden_states, ...)
        hidden_states += residual
        
        # Pre-Norm FFN + Residual
        hidden_states = hidden_states + self.mlp(
            self.post_attention_layernorm(hidden_states)
        )
        return hidden_states, present
```

FFN 根据 `use_moe` 自动选择 Dense (`FeedForward`) 或 MoE (`MOEFeedForward`)。

## GQA (Grouped Query Attention)

定义在 `modules/attention.py`，实现分组查询注意力：

| 参数 | 值 |
|------|-----|
| `num_attention_heads` (Q) | 8 |
| `num_key_value_heads` (KV) | 4 |
| `n_rep` (重复倍数) | 8/4 = 2 |

```
Q: [8 heads]  → 每组 2 个 Q 头共享 1 组 K/V
K: [4 heads]  → repeat_kv 扩展为 8 heads
V: [4 heads]  → repeat_kv 扩展为 8 heads
```

### Flash Attention 支持

当满足以下条件时使用 PyTorch 的 `scaled_dot_product_attention`:
- `config.flash_attn = True`
- `seq_len > 1`
- 非 causal 模式或首次前向（无 past_key_value）
- 无自定义 attention_mask

否则回退到手动实现的标准注意力。

### QK-Norm

每个 Attention 层在 RoPE 之前对 Q 和 K 分别应用 `RMSNorm`（`q_norm`、`k_norm`），这在 LLaMA 类模型中较为常见，有助于稳定训练。

## RoPE 与 YaRN 扩展

定义在 `modules/rope.py`：

### 标准 RoPE

```python
freqs = 1.0 / (rope_base ** (torch.arange(0, dim, 2) / dim))
freqs = torch.outer(t, freqs)
```

### YaRN 扩展

当 `inference_rope_scaling=True` 时启用 YaRN，可将上下文从 2048 扩展到 32768：

```python
ramp = clamp((arange - low) / (high - low), 0, 1)
freqs = freqs * (1 - ramp + ramp / factor)  # factor=16
```

- `beta_fast=32`, `beta_slow=1`: 控制哪些频段进行缩放
- `attention_factor=1.0`: 注意力 logits 缩放因子
- `original_max_position_embeddings=2048`: 原始预训练长度
